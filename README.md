## Chapter 01. 깨끗한 코드
* 주석없이 이해 가능한 코드
* **인력이 추가되더라도 생산성이 더 좋아지지 않은다.** 
* 원인은 나쁜코드
* 깨진 유리창 이론 - 문제가 발생했을 때 바로 고치지 않으면 금새 더 난장판 됨 
* **클린코드는 한가지 일을 잘한다.** 여러가지 일을 하려고 하지 않는다.
* 클린코드는 하나의 잘 쓰여진 산문처럼 읽혀야 한다. 가독성이 좋아야 한다.
* 수정하기 쉬워야 한다.
* TDD. 테스트 기반의 개발.
